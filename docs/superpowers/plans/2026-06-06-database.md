# 数据库与 Drizzle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立 Neon Postgres + Drizzle schema、db client、迁移配置和核心表 smoke tests。

**Architecture:** Schema 按业务边界拆分到 `src/lib/db/schema/*`，`drizzle.config.ts` 指向 schema 目录并输出迁移到 `drizzle/`，运行时使用 Neon HTTP driver。测试验证核心表和关键枚举导出，不连接真实数据库。

**Tech Stack:** Drizzle ORM, Drizzle Kit, Neon Serverless, TypeScript, Vitest.
